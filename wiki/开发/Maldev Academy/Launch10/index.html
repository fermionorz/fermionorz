模块 10 - 检测机制

目标
  • 了解 YARA 规则如何检测恶意软件
  • 在 VirusTotal 上搜索以下文件哈希：e8ac867e5f51bdcf5ab7b06a8bced131
  • 点击 VirusTotal 中文件结果中的“行为”选项卡。该文件执行了什么可疑/恶意行为？
  • 点击“详细信息”选项卡并向下滚动到“导入”。导入了哪些 DLL？从 User32.dll 导入了哪些 WinAPI？

介绍

安全解决方案使用多种技术来检测恶意软件。了解安全解决方案使用哪些技术来检测或将软件分类为恶意软件是很重要的。

静态/签名检测

签名是在恶意软件中唯一标识其的字节或字符串的数量。还可以指定其他条件，例如变量名和导入的函数。一旦安全解决方案扫描一个程序，它会试图将其与已知规则列表进行匹配。这些规则必须预先构建并推送到安全解决方案。YARA 是安全供应商用于构建检测规则的工具之一。例如，如果一个 shellcode 包含以 FC 48 83 E4 F0 E8 C0 00 00 00 41 51 41 50 52 51 开头的字节序列，那么这可以用来检测有效负载是 Msfvenom 的 x64 exec 有效负载。同样的检测机制可以用于文件中的字符串。

签名检测容易绕过，但可能会耗费时间。避免在恶意软件中硬编码可以唯一识别实现的值非常重要。本课程中提供的代码试图避免可能被硬编码的值，而是动态获取或计算这些值。

哈希检测

哈希检测是静态/签名检测的一个子集。这是一种非常直接的检测技术，是安全解决方案检测恶意软件的最快和最简单的方法。此方法仅需将已知恶意软件的哈希（例如 MD5，SHA256）保存到数据库中。恶意软件的文件哈希将与安全解决方案的哈希数据库进行比较，以查看是否存在匹配。

规避哈希检测非常简单，尽管可能单靠这一点不足够。通过更改文件中的至少一个字节，文件哈希会根据任何哈希算法发生变化，因此该文件将具有一个可能唯一的文件哈希。

启发式检测

由于签名检测方法可以通过对恶意文件进行小的更改轻松规避，因此引入了启发式检测，以识别在未知、新型和现有恶意软件的修改版本中可能存在的可疑特征。根据安全解决方案，启发式模型可以包含以下一种或两种内容：

  • 静态启发式分析 - 涉及反编译可疑程序并将代码片段与已知的恶意软件进行比较，这些恶意软件已经存在于启发式数据库中。如果源代码的特定百分比与启发式数据库中的任何内容匹配，则该程序将被标记。
  • 动态启发式分析 - 程序被放置在虚拟环境或沙箱中，然后由安全解决方案分析以寻找任何可疑行为。

动态启发式分析（沙箱检测）

沙盒检测通过在沙盒环境中执行文件动态分析文件的行为。在执行文件时，安全解决方案会寻找可疑的行为或被分类为恶意的行为。例如，分配内存不一定是恶意行为，但分配内存、联网获取 shellcode、将 shellcode 写入内存并按该顺序执行则被认为是恶意行为。

恶意软件开发人员将嵌入反沙箱技术以检测沙箱环境。如果恶意软件确认它是在沙箱中执行，则执行良性代码，否则执行恶意代码。

基于行为的检测

一旦恶意软件运行，安全解决方案将继续监控运行进程所执行的可疑行为。安全解决方案将查找可疑指示，例如加载 DLL、调用特定的 Windows API 和连接互联网。一旦检测到可疑行为，安全解决方案将对运行进程进行内存扫描。如果该进程被确定为恶意，将被终止。

某些操作可能会立即终止该过程，而不进行内存扫描。例如，如果恶意软件对 notepad.exe 进行进程注入并连接到互联网，这可能会导致该过程被立即终止，因为这很可能是恶意活动。

避免基于行为的检测的最佳方法是使过程尽可能表现得无害（例如，避免生成 cmd.exe 子进程）。此外，可以通过内存加密来规避内存扫描。这是一个更高级的话题，将在未来的模块中讨论。

API Hooking

API hooking 是一种由安全解决方案（主要是 EDR）使用的技术，用于实时监控进程或代码执行的恶意行为。API hooking 通过拦截常被滥用的 API，然后实时分析这些 API 的参数来工作。这是一种强大的检测方式，因为它允许安全解决方案在 API 被去混淆或解密后，查看传递给 API 的内容。这种检测被认为是实时和基于行为检测的结合。

IAT 检查

在 PE 结构中讨论的一个组件是导入地址表（IAT）。简要总结一下 IAT 的功能，它包含在运行时使用的函数名称。它还包含导出这些函数的库（DLL）。这条信息对于安全解决方案非常有价值，因为它知道可执行文件使用了哪些 WinAPI。

例如，勒索软件用于加密文件，因此它很可能会使用加密和文件管理功能。当安全解决方案看到 IAT 包含这些类型的函数，例如 CreateFileA/W, SetFilePointer, Read/WriteFile, CryptCreateHash, CryptHashData, CryptGetHashParam 时，程序要么被标记，要么会受到额外的审查。下面的图像显示了 dumpbin.exe 工具用于检查二进制文件的 IAT。

手动分析

尽管绕过了上述所有检测机制，蓝队和恶意软件分析师仍可以手动分析恶意软件。一位精通恶意软件反向工程的防御者可能能够检测到恶意软件。此外，安全解决方案通常会将可疑文件的副本发送到云端进行进一步分析。

恶意软件开发者可以实施反逆向工程技术，使逆向工程的过程更加困难。一些技术包括调试器的检测和虚拟化环境的检测，这将在未来的模块中讨论。
