模块 11 - Windows 进程

目标
  • 打开 Process Hacker，双击一个进程，选择“线程”标签，并查看正在运行的线程数量
  • 点击“内存”标签页，查看不同的内存区域
  • 请阅读微软关于 OpenProcess 和 OpenThread 的文档
  • 可选：运行 notepad.exe，然后使用 OpenProcess 获取该进程的句柄（提示：从 Process Hacker 中复制记事本的 PID）
  • 查看不同的内存类型
  • 审查 PEB 和 TEB 结构

什么是 Windows 进程？

Windows 进程是运行在 Windows 机器上的程序或应用。进程可以由用户或系统本身启动。进程消耗资源，如内存、磁盘空间和处理器时间，以完成任务。

进程 - 线程

Windows 进程由一个或多个线程组成，这些线程都是并发运行的。线程是一组可以在进程内独立执行的指令。进程内的线程可以相互通信和共享数据。线程由操作系统调度执行，并在进程的上下文中进行管理。

进程 - 内存

Windows 进程也使用内存来存储数据和指令。创建进程时会分配内存，分配的数量可以由进程自身设置。操作系统通过使用虚拟内存和物理内存来管理内存。虚拟内存允许操作系统使用超过物理可用内存的更多内存，通过创建一个可被应用程序访问的虚拟地址空间。这些虚拟地址空间被划分成"页面"，然后分配给进程。

内存类型

进程可以有不同类型的内存：

  • 私有内存专用于单个进程，无法被其他进程共享。这种类型的内存用于存储特定于该进程的数据。
  • 映射内存可以由两个或多个进程共享。它用于在进程之间共享数据，比如共享库、共享内存段和共享文件。映射内存对于其他进程是可见的，但不能被其他进程修改。
  • 镜像内存包含可执行文件的代码和数据。它用于存储进程使用的代码和数据，比如程序的代码、数据和资源。镜像内存通常与加载到进程地址空间的 DLL 文件相关。

进程环境块（PEB）

进程环境块（PEB）是 Windows 中的一种数据结构，包含有关进程的信息，例如其参数、启动信息、分配的堆信息和已加载的 DLL 等。操作系统用于存储正在运行的进程的信息，Windows 加载器用于启动应用程序。它还存储有关进程的信息，例如进程 ID（PID）和可执行文件的路径。

每个创建的进程都有其自己的 PEB 数据结构，包含有关其自身的一组信息。

PEB 结构体

在 C 语言中，PEB 结构如下所示。此结构的保留成员可以忽略。

typedef struct _PEB {
  BYTE                          Reserved1[2];
  BYTE                          BeingDebugged;
  BYTE                          Reserved2[1];
  PVOID                         Reserved3[2];
  PPEB_LDR_DATA                 Ldr;
  PRTL_USER_PROCESS_PARAMETERS  ProcessParameters;
  PVOID                         Reserved4[3];
  PVOID                         AtlThunkSListPtr;
  PVOID                         Reserved5;
  ULONG                         Reserved6;
  PVOID                         Reserved7;
  ULONG                         Reserved8;
  ULONG                         AtlThunkSListPtr32;
  PVOID                         Reserved9[45];
  BYTE                          Reserved10[96];
  PPS_POST_PROCESS_INIT_ROUTINE PostProcessInitRoutine;
  BYTE                          Reserved11[128];
  PVOID                         Reserved12[1];
  ULONG                         SessionId;
} PEB, *PPEB;

非保留成员如下所述。

BeingDebugged

BeingDebugged 是 PEB 结构中的一个标志，用于指示进程是否正在被调试。当进程正在被调试时，它被设置为 1 (TRUE)，而当没有被调试时，它被设置为 0 (FALSE)。 Windows 加载器使用它来确定是否附加调试器来启动应用程序。

加载器 (Ldr)

Ldr 是指向进程环境块 (PEB) 中的 PEB_LDR_DATA 结构的指针。该结构包含有关进程加载的动态链接库 (DLL) 模块的信息。它包括在进程中加载的 DLL 列表、每个 DLL 的基地址以及每个模块的大小。Windows 加载程序使用它来跟踪在进程中加载的 DLL。 PEB_LDR_DATA 结构如下所示。

typedef struct _PEB_LDR_DATA {
  BYTE       Reserved1[8];
  PVOID      Reserved2[3];
  LIST_ENTRY InMemoryOrderModuleList;
} PEB_LDR_DATA, *PPEB_LDR_DATA;

Ldr 可以用来找到特定 DLL 的基本地址，以及该内存空间中驻留的函数。这将在未来的模块中用于构建自定义版本的 GetModuleHandleA/W，以增加隐蔽性。

ProcessParameters

ProcessParameters 是 PEB 中的数据结构。它包含在创建进程时传递给进程的命令行参数。Windows 加载器将这些参数添加到进程的 PEB 结构中。ProcessParameters 是指向下面显示的 RTL_USER_PROCESS_PARAMETERS 结构的指针。

typedef struct _RTL_USER_PROCESS_PARAMETERS {
  BYTE           Reserved1[16];
  PVOID          Reserved2[10];
  UNICODE_STRING ImagePathName;
  UNICODE_STRING CommandLine;
} RTL_USER_PROCESS_PARAMETERS, *PRTL_USER_PROCESS_PARAMETERS;

ProcessParameters 将在未来的模块中被利用，以执行诸如命令行伪装的操作。

AtlThunkSListPtr & AtlThunkSListPtr32

AtlThunkSListPtr 和 AtlThunkSListPtr32 被 ATL（活动模板库）模块用于存储指向 thunk 函数链表的指针。Thunk 函数用于调用在不同地址空间中实现的函数，这些函数通常代表从 DLL（动态链接库）文件中导出的函数。ATL 模块使用 thunk 函数的链表来管理 thunk 过程。

PostProcessInitRoutine

PEB 结构中的 PostProcessInitRoutine 字段用于存储一个指向函数的指针，该函数在操作系统完成进程中所有线程的 TLS（线程本地存储）初始化后被调用。此函数可用于执行进程所需的任何额外初始化任务。

TLS 和 TLS 回调将在需要时更详细地讨论。

SessioinId

PEB 中的 SessionID 是分配给单个会话的唯一标识符。它用于跟踪用户在会话期间的活动。

线程环境块 (TEB)

线程环境块（TEB）是 Windows 中的一种数据结构，用于存储有关线程的信息。它包含线程的环境、安全上下文以及其他相关信息。它存储在线程的堆栈中，并由 Windows 内核用于管理线程。

TEB 结构体

TEB 结构在 C 中如下所示。该结构的保留成员可以忽略。

typedef struct _TEB {
  PVOID Reserved1[12];
  PPEB  ProcessEnvironmentBlock; // 进程环境块
  PVOID Reserved2[399];
  BYTE  Reserved3[1952];
  PVOID TlsSlots[64];  // 线程局部存储 (TLS) 槽
  BYTE  Reserved4[8];
  PVOID Reserved5[26];
  PVOID ReservedForOle; // 为 OLE 保留
  PVOID Reserved6[4];
  PVOID TlsExpansionSlots; // TLS 扩展槽
} TEB, *PTEB;

进程环境块 (PEB)

指向上述 PEB 结构的指针，PEB 位于线程环境块（TEB）内部，用于存储有关当前运行进程的信息。

TlsSlots

TLS（线程局部存储）槽是 TEB 中用于存储线程特定数据的位置。Windows 中的每个线程都有自己的 TEB，每个 TEB 具有一组 TLS 槽。应用程序可以使用这些槽来存储该线程特定的数据，例如线程特定变量、线程特定句柄、线程特定状态等。

Tls 扩展槽（TlsExpansionSlots）

TEB 中的 TLS 扩展槽是一组指针，用于存储线程的线程局部存储数据。TLS 扩展槽保留供系统 DLL 使用。

进程和线程句柄

在 Windows 操作系统中，每个进程都有一个独特的进程标识符或进程 ID（PID），该标识符由操作系统在进程创建时分配。PID 用于区分一个运行中的进程与另一个运行中的进程。相同的概念适用于运行中的线程，运行中的线程有一个唯一的 ID，用于将其与系统中其他现有线程（在任何进程中）区分开。

这些标识符可以用来通过下面的 WinAPI 打开对进程或线程的句柄。

  • OpenProcess - 通过标识符打开一个现有的进程对象句柄。
  • OpenThread - 通过标识符打开一个现有的线程对象句柄。

这些 WinAPI 将在后续需要时进一步详细讨论。目前，了解打开的句柄可以用于对其相关的 Windows 对象执行进一步操作，例如挂起进程或线程，已经足够了。

句柄在不再需要使用时应该始终关闭，以避免 句柄泄漏。这是通过 CloseHandle WinAPI 调用实现的。
