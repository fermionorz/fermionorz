模块 15 - Payload 部署 - .rsrc 段

目标
  • 按照模块中的步骤，在资源部分存储有效载荷

介绍

将有效负载保存在 .rsrc 部分是最佳选择之一，因为大多数实际的二进制文件都会在此处保存其数据。这对于恶意软件作者来说也是一种更清晰的方法，因为较大的有效负载无法存储在 .data 或 .rdata 部分，由于大小限制，这会导致在编译期间 Visual Studio 出现错误。

.rsrc 节

以下步骤说明如何在 .rsrc 部分存储有效载荷。

  1. 在 Visual Studio 中，右键单击“资源文件”，然后单击添加 > 新项。

  1. 点击“资源文件”。

  1. 这将生成一个新的侧边栏，即资源视图。右键单击 .rc 文件（Resource.rc 是默认名称），并选择“添加资源”选项。

  1. 点击“导入”。

  1. 选择 calc.ico 文件，这是重命名为 .ico 扩展名的原始有效负载。

  1. 会出现一个提示，要求输入资源类型。输入 "RCDATA"（不带引号）。

  1. 点击确定后，有效载荷应以原始二进制格式显示在 Visual Studio 项目中。

  1. 当退出资源视图时，应该可以看到“resource.h”头文件，并根据第 2 步中的.rc 文件命名。该文件包含一个定义语句，引用资源部分中的有效载荷 ID（IDR_RCDATA1）。这是为了能够在后续从资源部分检索有效载荷而重要的。

一旦编译，负载将存储在 .rsrc 部分，但无法直接访问。相反，必须使用多个 WinAPI 来访问它。

  • FindResourceW - 获取特定 ID 传递的资源部分中存储的指定数据的位置（这在头文件中定义）
  • LoadResource - 检索资源数据的 HGLOBAL 句柄。此句柄可用于获取指定资源在内存中的基地址。
  • LockResource - 从其句柄中获取指向资源部分中指定数据的指针。
  • SizeofResource - 获取资源部分中指定数据的大小。

以下代码片段将利用上述 Windows API 访问 .rsrc 部分并获取有效负载地址和大小。

#include <Windows.h>
#include <stdio.h>
#include "resource.h"

int main() {

    HRSRC       hRsrc                   = NULL;
    HGLOBAL     hGlobal                 = NULL;
    PVOID       pPayloadAddress         = NULL;
    SIZE_T      sPayloadSize            = NULL;

    // 根据其 ID *IDR_RCDATA1* 获取存储在 .rsrc 中的数据的位置
    hRsrc = FindResourceW(NULL, MAKEINTRESOURCEW(IDR_RCDATA1), RT_RCDATA);
    if (hRsrc == NULL) {
        // 函数发生错误
        printf("[!] FindResourceW 失败，错误信息：%d \n", GetLastError());
        return -1;
    }

    // 后续调用 LockResource，因此需要获取 HGLOBAL 或指定资源数据的句柄
    hGlobal = LoadResource(NULL, hRsrc);
    if (hGlobal == NULL) {
        // 函数发生错误
        printf("[!] LoadResource 失败，错误信息：%d \n", GetLastError());
        return -1;
    }

    // 获取 .rsrc 部分中有效负载的地址
    pPayloadAddress = LockResource(hGlobal);
    if (pPayloadAddress == NULL) {
        // 函数发生错误
        printf("[!] LockResource 失败，错误信息：%d \n", GetLastError());
        return -1;
    }

    // 获取 .rsrc 部分中有效负载的尺寸
    sPayloadSize = SizeofResource(NULL, hRsrc);
    if (sPayloadSize == NULL) {
        // 函数发生错误
        printf("[!] SizeofResource 失败，错误信息：%d \n", GetLastError());
        return -1;
    }

    // 在屏幕上打印指针和尺寸
    printf("[i] pPayloadAddress 变量：0x%p \n", pPayloadAddress);
    printf("[i] sPayloadSize 变量：%ld \n", sPayloadSize);
    printf("[#] 按下 <Enter> 退出...");
    getchar();
    return 0;
}

在编译和运行上述代码后，有效负载地址及其大小将打印到屏幕上。重要的是要注意，这个地址位于 .rsrc 节中，该节是只读内存，任何尝试更改或编辑其中数据的操作都会导致访问冲突错误。要编辑有效负载，必须分配一个与有效负载大小相同的缓冲区并进行复制。这个新缓冲区就是可以进行更改（如解密有效负载）的地方。

更新 .rsrc 段的 Payload

由于有效负载无法直接从资源部分编辑，因此必须将其移动到一个临时缓冲区。为此，使用 HeapAlloc 分配 Payload 大小的内存，然后使用 memcpy 将有效负载从资源部分移动到临时缓冲区。

// 使用 HeapAlloc 调用分配内存
PVOID pTmpBuffer = HeapAlloc(GetProcessHeap(), 0, sPayloadSize);
if (pTmpBuffer != NULL) {
    // 将有效负载从资源部分复制到新缓冲区
    memcpy(pTmpBuffer, pPayloadAddress, sPayloadSize);
}

// 打印缓冲区 (pTmpBuffer) 的基址
printf("[i] pTmpBuffer var : 0x%p \n", pTmpBuffer);

由于 pTmpBuffer 现在指向一个可写的内存区域，该区域保存着有效负载，因此可以解密有效负载或对其进行任何更新。
